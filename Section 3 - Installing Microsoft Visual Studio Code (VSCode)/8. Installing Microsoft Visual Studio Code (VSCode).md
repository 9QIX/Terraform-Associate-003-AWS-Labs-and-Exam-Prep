# Installing Microsoft Visual Studio Code (VS Code)

## Overview

This lesson covers the installation and configuration of Visual Studio Code (VS Code), the recommended IDE (Integrated Development Environment) for Terraform development. VS Code provides syntax highlighting, extensions for HCL (HashiCorp Configuration Language), integrated terminal access, and source control integration.

---

## Concepts

### Visual Studio Code (VS Code)

- **Free, open-source IDE** developed by Microsoft
- Cross-platform support: Windows, macOS, Linux
- Extensible via marketplace extensions
- Built-in terminal, Git integration, and file explorer

### Key Features for Terraform Development

- **Syntax highlighting** for HCL files (`.tf`, `.tfvars`)
- **Extensions** for Terraform, Docker, Kubernetes, Python, and more
- **Integrated terminal** for running Terraform commands without leaving the IDE
- **Source control integration** for Git/GitHub workflows
- **Workspace management** for organizing multiple project folders

---

## Installation Procedures

### Step 1: Download VS Code

1. Navigate to [code.visualstudio.com/download](https://code.visualstudio.com/download)
2. Select the appropriate installer for your operating system:
   - **Windows** — `.exe` installer
   - **macOS** — Universal or Apple Silicon (M1/M2/M3) or Intel builds
   - **Linux** — `.deb` (Debian/Ubuntu), `.rpm` (Red Hat/Fedora/SUSE), or Snap package
3. Download and run the installer
4. Follow the installation wizard (default settings recommended)

### Platform-Specific Downloads

| Platform    | Download Options                               |
| ----------- | ---------------------------------------------- |
| **Windows** | User Installer, System Installer, Portable ZIP |
| **macOS**   | Universal, Apple Silicon, Intel                |
| **Linux**   | `.deb`, `.rpm`, Snap, `.tar.gz`                |

## Workspace Configuration

### Opening a Project Folder

1. Click **Explorer** icon (top-left sidebar)
2. Click **Open Folder**
3. Navigate to your Terraform project directory (e.g., `~/terraform/demo`)
4. Click **Select Folder** (macOS) or **Open** (Windows)

**Result:**

- All files in the directory appear in the Explorer pane
- Terraform files (`.tf`, `.tfvars`, `.tfstate`) are visible and editable

### Adding Multiple Folders to Workspace

1. Right-click in the Explorer pane
2. Select **Add Folder to Workspace**
3. Choose additional directories to include

**Use Case:**

- Manage multiple Terraform projects simultaneously
- Reference shared modules across projects

## Integrated Terminal

### Opening the Terminal

**Method 1: Right-Click Context Menu**

1. Right-click any file or folder in Explorer
2. Select **Open in Integrated Terminal**

**Method 2: Menu Bar**

- **Terminal** → **New Terminal**

**Method 3: Keyboard Shortcut**

- **Windows/Linux:** `Ctrl + `` (backtick)
- **macOS:** `Cmd + `` (backtick)

### Terminal Features

```bash
# Terminal opens in the current workspace directory
pwd
# Output: /Users/BK/terraform/demo

# Run Terraform commands directly
terraform init
terraform plan
terraform apply
```

**Line-by-line Comments:**

```bash
# Display current working directory
pwd

# Initialize Terraform working directory
terraform init

# Generate execution plan
terraform plan

# Apply infrastructure changes
terraform apply
```

**Code Walkthrough:**

- Terminal automatically sets the working directory to the opened folder
- No need to manually `cd` into the project directory
- Multiple terminals can be opened simultaneously (tabs appear in the terminal pane)

### Managing Multiple Terminals

- Click the **+** icon in the terminal pane to open additional terminals
- Switch between terminals using the dropdown menu (top-right of terminal pane)
- Each terminal maintains its own session and working directory

## Essential Extensions for Terraform

### Recommended Extensions

| Extension               | Publisher | Purpose                                                       |
| ----------------------- | --------- | ------------------------------------------------------------- |
| **HashiCorp Terraform** | HashiCorp | Syntax highlighting, IntelliSense, validation for `.tf` files |
| **HashiCorp HCL**       | HashiCorp | HCL syntax support and formatting                             |
| **GitHub Copilot**      | GitHub    | AI-powered code completion (requires subscription)            |
| **Docker**              | Microsoft | Docker container management                                   |
| **Kubernetes**          | Microsoft | Kubernetes resource management                                |
| **CodeSnap**            | adpyke    | Screenshot code snippets for documentation                    |
| **Python**              | Microsoft | Python language support (for Terraform scripts)               |

### Installing Extensions

1. Click **Extensions** icon (left sidebar)
2. Search for the extension name (e.g., "HashiCorp Terraform")
3. Click **Install**
4. Reload VS Code if prompted

**Example: Installing HashiCorp Terraform Extension**

```bash
# Search in Extensions Marketplace
Search: "HashiCorp Terraform"

# Click Install on:
# - HashiCorp Terraform (official extension)
# - HashiCorp HCL (syntax support)
```

**Code Walkthrough:**

- Extensions add new icons to the left sidebar (e.g., Terraform icon for HCP Terraform integration)
- Some extensions require additional configuration (e.g., GitHub Copilot requires authentication)
- Extensions auto-update by default

## Source Control Integration

### Git/GitHub Features

- **View changes** in the Source Control pane
- **Stage, commit, and push** changes without leaving VS Code
- **Diff viewer** for comparing file versions
- **Branch management** and merge conflict resolution

### Workflow Example

1. Make changes to Terraform files
2. Click **Source Control** icon (left sidebar)
3. Review changed files
4. Enter commit message
5. Click **Commit** → **Push** to remote repository

**No separate Git client required** — VS Code handles all Git operations natively.

## Customization

### Themes

1. **File** → **Preferences** → **Color Theme** (Windows/Linux)
2. **Code** → **Preferences** → **Color Theme** (macOS)
3. Select from built-in themes or install custom themes from Extensions Marketplace

**Popular Themes:**

- Dark+ (default dark)
- Light+ (default light)
- Monokai
- Dracula

### Settings

- **File** → **Preferences** → **Settings** (Windows/Linux)
- **Code** → **Preferences** → **Settings** (macOS)

**Common Settings for Terraform:**

- Auto-save: `Files: Auto Save` → `afterDelay`
- Format on save: `Editor: Format On Save` → Enable
- Tab size: `Editor: Tab Size` → `2` (HCL convention)

## Pitfalls & Best Practices

### Best Practices

- **Install HashiCorp Terraform extension** — Provides syntax validation and auto-completion
- **Use integrated terminal** — Eliminates context switching between IDE and terminal
- **Enable auto-save** — Prevents loss of unsaved changes
- **Organize workspaces** — Group related Terraform projects for easier navigation
- **Leverage Git integration** — Commit frequently and track infrastructure changes

### Common Pitfalls

- **Forgetting to install HCL extension** — Results in no syntax highlighting for `.tf` files
- **Not reloading after extension installation** — Some extensions require VS Code restart
- **Ignoring terminal working directory** — Always verify `pwd` before running Terraform commands
- **Overloading with extensions** — Too many extensions can slow down VS Code performance
